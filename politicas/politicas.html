Políticas de Impulse

Objetivo

Especificar qué acciones se deben realizar siempre mientras se trabaja.

  • 1: Manejo de configuración

    • 1.1: Siempre que se termine un producto de trabajo, evaluar si es un elemento de configuración siguiendo los criterios.
    • 1.2: Si se identifica un elemento de configuración, debe añadírsele la información pertinente (guía de formato).
    • 1.3: Siempre que se edite un configuration item, seguir guía para hacer pull requests.
    • 1.4: Cuando haya una solicitud de cambio, realizar una revisión estática (guía) y asegurar que se hayan corrido las pruebas
