Guía para hacer Pull Requests

Objetivo

        

Explicar a los integrantes del departamento cómo hacer solicitudes de cambio para los elementos de configuración, con el propósito de prevenir defectos, mejorar la calidad de los productos de trabajo, lograr trazabilidad entre los cambios y permitir la gestión de versiones.

Nota:

Los cambios a realizar no forzosamente tienen que ser código.

Prerrequisitos para hacer una solicitud de cambios:

  1. Asegúrate de que lo que deseas integrar sea una tarea completa.
  2. Asegúrate de que los cambios hayan cumplido con las checklists de calidad que les corresponda.
  3. Asegúrate de que lo que estés subiendo a nivel controlado sea un elemento de configuración (https://impulse-semestrei.github.io/checklists/ChecklistConfiguracion.html).

Pasos para hacer una solicitud de cambios:

  1. Añadir los cambios
  1. Ver los cambios que están en la versión que estás editando.  Al usar git, puedes emplear el comando “git status” para ver el cambio en tu rama.
  2. Obtener otros cambios realizados en el elemento de configuración. En git, puedes obtener los cambios nuevos usando el comando “git pull”.
  3. Revisar los cambios a realizar.
  4. Añadir los cambios necesarios a la solicitud de cambio. Si estás usando git y deseas incluir todos los cambios dentro de tu solicitud, ejecuta el comando “git add -A”. Si hay cambios que no deseas incluir, haz “git add <path>” por cada uno de los cambios que deseas incluir en tu solicitud.
  5. Eliminar cambios que no sean necesarios. En git, después de haber incluido los cambios ejecuta el comando “git stash” para eliminar los cambios innecesarios de tu rama.
  1. Guardar los cambios en el sistema de manejo de configuración
  1. En tu solicitud, añade un mensaje que explique los cambios realizados. Si estás usando git y los cambios pueden expresarse en una sola línea, usa el comando “git commit -m <mensaje>”. Si hiciste una gran cantidad de cambios que no pueden expresarse en una sola línea, haz un listado de los cambios en. corre el comando “git commit”(sin el “-m”) para abrir un editor de texto.
  1. Dentro del editor, utiliza la primera línea para escribir el asunto del cambio. El asunto debe contener un verbo en infinitivo y un resumen de los cambios.
  2. Escribe a continuación cada uno de los cambios en bullet points de manera más específica.
  1. Publica los cambios al sistema de manejo de configuración. Si estás usando git, ejecuta el comando “git push origin <tu_rama>”.
  2. Obtén la última versión del nivel controlado del sistema de configuración. Si estás usando git, ejecuta el comando “git pull origin <rama a la que se desea integrar>”. Realizar luego lo siguiente:
  1. Si hay conflictos, resuélvelos.
  2. Añade los cambios realizados. Si estás en git, ejecuta el comando “git add -A”.
  3. Añade un mensaje que explique los cambios. En git, ejecuta el comando “ git commit -m ‘arreglar conflictos’  ”.
  1. Publica los cambios en el sistema de manejo de configuración. En git, ejecuta el comando “git push origin <tu_rama>”.

Hacer una solicitud de cambio.

Si estás usando git, hacer lo siguiente:

  1. Abre en el navegador el repositorio de github correspondiente.
  2. Dirígete a  la pestaña de “Pull Requests”.
  3. Da clic en el botón “New pull request”.
  4. Deja la rama a la que quieres integrar los cambios como rama base y pon como rama de comparación la rama que deseas integrar.
  5. Da clic al botón “Create new pull request”.
  6. Revisa que el nombre del pull request sea el correcto.
  7. Haz un listado de todos los cambios a aplicar en el cuerpo del pull request.
  8. Da click en el apartado de Reviewers del lado derecho de la página
  9. Escoger a un reviewer
  1. Si no sabes a quién escoger, puedes utilizar el inventario de habilidades para encontrar a un reviewer adecuado.
  2. Si deseas integrar a master, incluye como reviewers a los integrantes del CCB.
  1. Da clic en el botón “Create pull request”.
