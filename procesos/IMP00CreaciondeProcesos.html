Creación de Procesos

Propósito

  • Tener una forma de trabajo que nos permita tener resultados replicables

Criterios de entrada

  • Mapa de procesos a la mano
  • Tener un medio visual accesible
  • Tener la propuesta para cambiar en la forma de trabajo
  • Tener el CMMI disponible
  • Tener un total conocimiento de la política de obtención de d atos.

P.A
1.- Categoría
  • Definir si lo que se va a hacer es un proceso, guía o política (Guía)
  • Si es guía ver template y terminar proceso
  • Si es política ver template y terminar proceso
 PMC 2.2
OPD 1.1
2.- Definir propósito
  • Identificar la necesidad del cambio
  • Identificar el valor aportado del cambio a la forma de trabajo
  • Hacer la pregunta ¿Por qué? y ¿Para qué? (sugerencia: realizarlas varias veces)
  • Definir un nombre acorde al propósito
 OPF 1.1
PMC 2.1 / 1.4
3.- Definir resultados
  • Analizar el mapa de procesos para conocer qué procesos dependen del nuevo.
  • Identificar la herramienta, documento o artefacto que mejor se acople al propósito.
  • Verificar que la salida sea replicable.
  • Definir formato y ubicación del resultado que será usado como evidencia
 OPF 1.1
PMC 1.4
4.- Definir entradas
  • Analizar el mapa de procesos para conocer qué procesos dependen del nuevo.
  • Definir las características de la entrada
  • Verificar que las entradas sean útiles, valiosas y compatibles para el resultado
 OPD 1.3
5.- Construcción del procesos
  • Definir fases. Son las etapas del proceso
  • Definir pasos. Son las acciones atómicas a realizar en cada etapa
  • Definir áreas de procesos. Asociar las áreas del CMMI que cubre cada una de las fases
 OPD 1.1
6.- Revisión del proceso
  • Leer el proceso de nuevo y buscar defectos (faltas ortográficas, ambigüedades, partes incompletas, congruencia, etc...)
  • Obtener el compromiso de aquellos que realizan el proceso
