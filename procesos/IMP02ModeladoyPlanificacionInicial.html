DEP10 Modelado y Planificación inicial del proyecto

Propósito

  • Asegurar el alcance del proyecto.
  • Establecer y mantener planes que definan las actividades del proyecto.
  • Asegurar las acciones para alcanzar el objetivo del proyecto.

Criterios de entrada

  • Haber seleccionado previamente un proyecto.
  • Tener presente la misión y visión del departamento.
  • Conocer los objetivos y necesidades del proyecto.

Fases

Fase

Pasos

P.A
1.- Familiarizarse con WoW de Impulse.
  • Familiarizarse con el mapa de procesos y sus respectivos triggers, resultados y fases.
  • Conocer los recursos que se utilizan como templates, IDE's u otras licencias.
  • Entender la ubicación de tu equipo en el Organigrama de Impulse y la relación con el socio formador y demas stakeholders (conocimiento de otros equipos, comites, project managers y auditores).
  • PP 2.3/ 2.6/ 3.1
2.- Preparar estructura de directorio de trabajo
  • Crear una carpeta que se llame "Fase de inicio" dentro de el directorio de tu proyecto.Dentro de aqui se almacenaran todos los resultados de trabajo de la fase de inicio
3.- Diseñar requisitos
  • Crear carpeta llamada "Diseñar requisitos" y guardar en esta carpeta todos los archivos generados por esta fase
  • Entender el Business Process Diagram de la empresa
  • Crear un glosario de los términos de la empresa
  • Definir requsitos funcionales
  • Definir requisitos no funcionales
  • PP 2.6
  • RD 1.2
4.- Definir estrategia tecnica inicial
  • Crear carpeta llamada "Estrategia Inicial" y guardar en esta carpeta todos los archivos generados por esta fase
  • Definir el Technology Stack Diagram
  • Definir la User Interface Flow Diagram
  • Definir el Logical Data Model
  • Definir el Deployment Diagram
  • PP 2.6 / 2.3 / 2.7
  • RD 1.1
5.- Formación del equipo
  • Formar equipo con base en inventario de hablidiades(link a guía de inventario de habilidades)
  • PP 2.5
6.- Definir plan de liberación
  • Crear una carpeta llamada "Plan de liberación" y guardar en esta carpeta irán todos los archivos generados por esta fase
  • Asignar los agile points a requerimientos
  • Priorizar los requerimientos
  • Definir el WBS
  • Definir las estimaciones con base en actividades del WBS
  • Establecer el horario de trabajo de impulse
  • Definir el plan de liberación
  • Definir los objetivos de cada liberación
 PP 2.6/ 1.1/ 1.2/ 1.3/ 1.4/ 2.7/ 2.1
7.- Comunicar visión del proyecto
  • Agendar una cita con cliente
  • Presentar los requisitos iniciales
  • Presentar la estrategia tecnologica inicial
  • Presentar el plan de liberación
  • PP 3.2/ 3.3
  • RD 1.1
8.- Asegurar financiamiento
  • Presentar el alcance del plan inicial de proyecto
  • Negociar el alcance del plan inicial de proyecto
  • Establecer presupuesto con base en al alcance del proyecto
  • Firmar compromiso de alcance
  • PP 2.6 /2.1/ 3.2 /3.3
  • RD 1.1
9.- Identificación de riesgos
  • Referirse al proceso de identificación de riesgos
  • PP 2.5
10.- Preparación de ambiente trabajo
  • Asegurarse de que el 90% de los elementos de checklist se encuentran dentro del área de trabajo
  • Instalar herramientas de desarrollo con base en technology stack diagram
  • PP 2.4

Resultado

  • Un nuevo WBS.
  • Plan de liberación inicial definido.
  • Requisitos iniciales establecidos.
  • Documento de riesgos identificados.
  • Checklist de los recursos necesarios para llevar a cabo el proyecto.
  • Burndown chart del proyecto (horas y Agile Points)
  • Elementos de configuración establecidos:
    • Historias de usuario.
    • Plan de liberación inicial definido.
    • Diagrama de arquitectura.
    • Technology stack diagram.
