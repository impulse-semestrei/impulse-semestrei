Políticas

Política de revisión de datos

Objetivo

Definir las reglas para la revisión de datos de forma válida e íntegra que permitan su posterior análisis.

  • 1. Responsables

    • 1.1 El rol responsable de realizar el análisis de datos también verificará la integridad de los mismos.
    • 1.2 El responsable puede asignar a alguien más para que valide los datos.
    • 1.3 En casos de fuerza mayor, el Team Leader o Program Manager pueden reasignar al responsable.

  • 2. Criterios de revisión

    El responsable de la verificación de datos hará uso de la siguiente lista de criterios para asegurar que sean válidos e íntegros:

    • 2.1 Las personas que obtuvieron los datos deben también registrarlos.
    • 2.2 Los datos deben registrarse el mismo día que se obtuvieron.
    • 2.3 Los datos no deben estar incompletos.
    • 2.4 Los grupos de datos no deben tener patrones sospechosos.
      • Por ejemplo: Si en los timelogs hay demasiadas horas de inicio o fin que tengan los minutos terminados en 5 o 0 (como 6:15 o 6:10), se pueden considerar como datos corruptos.
    • 2.5 Los datos que incumplan con los puntos anteriores se considerarán como corruptos.
      • Por ejemplo: Si en el timelog hay una hora de inicio para una actividad, pero no una de fin, y el dato se registró el día pasado, se considerará como dato corrupto.

  • 3. Descarte de datos corruptos

    • 3.1 Los datos corruptos deben ser elimiados del registro.
    • 3.2 La persona que encuentre el dato corrupto debe registrar en el Log de corrupción de datos todos los campos que van desde "# de error" hasta "Detectado por".
    • 3.3 La persona que realice las correcciones a los datos debe llenar, en el mismo log, desde los campos "Acción correctiva" hasta "Responsable".
      • La persona que realice las correcciones no necesita tener el mismo rol que el responsable de revisión.
      • No se considera como acción correctiva el borrar los datos corruptos ni llenar los datos faltantes. La acción correctiva debe ser algo que ayude a que, en el futuro, los siguientes datos del mismo tipo no tengan el mismo problema.
