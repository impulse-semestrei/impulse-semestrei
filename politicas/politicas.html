Políticas

Políticas de Impulse

Objetivo

Especificar qué acciones se deben realizar siempre mientras se trabaja.

  • 1: Manejo de configuración

    • 1.1: Siempre que se termine cualquier producto de trabajo (no necesariamente código), evaluar si es un elemento de configuración siguiendo los criterios.
    • 1.2: Si se identifica un elemento de configuración, debe añadírsele la información pertinente (guía de formato).
    • 1.3: Siempre que se edite un elemento de configuración, seguir guía para hacer pull requests.
    • 1.4: Cuando haya una solicitud de cambio, debe realizarse una revisión. Si se trata de código, hacer una revisión estática (guía) y asegurar que se hayan corrido las pruebas
    • 1.5: En cuanto se sepa de un problema en el nivel estático, debe regresarse tan pronto como sea posible a la versión anterior de ese mismo nivel e implementar soluciones para corregirlo (Si estás usando git, usa esta referencia).
    • 1.6: Si se encuentra un problema en el nivel controlado, evaluar si puede arreglarse en un tiempo razonable. De lo contrario, regresar a la versión anterior a los últimos cambios (Si estás usando git, usa esta referencia).
