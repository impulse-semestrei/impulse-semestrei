Descripción del sistema de manejo de la configuración

Objetivo

Informar a los miembros del equipo sobre la estructura del sistema de manejo de configuración para facilitar su uso.

Estructura del sistema de manejo de la configuración

Todos los elementos de configuración del departamento están en el sistema de manejo de la configuración. El sistema de manejo de configuración se compone de varios niveles: dinámico, controlado y estático.

El nivel dinámico no tiene ninguna restricción en cuanto a los cambios que pueden hacerse y no requiere revisiones.

El nivel controlado requiere como criterio de entrada que se haga una revisión de los cambios antes de integrar. Este nivel contiene los cambios ya revisados que requieren más seguimiento antes de considerarse estables.

El nivel estático contiene las versiones estables de los elementos de configuración del departamento. Una versión estable requiere que todos sus elementos estén integrados, funcionen correctamente y estén aprobados.

Flujo de cambios de elementos de configuración

Los cambios a un elemento de configuración siguen el siguiente flujo:

  • Primero, se realizan en el nivel dinámico.
  • Luego, se hace una solicitud de integración al nivel controlado
  • Los cambios que contiene la solicitud se revisan y, de ser aprobados, se integran con el nivel controlado.
  • En el nivel controlado se realizan pruebas y seguimiento a los elementos de configuración y sus cambios.
  • Una vez que han sido probados, se hace una solicitud para integrarlos con el nivel estático.
  • El Configuration Control Board se reúne para revisar la solicitud de cambio a nivel estático. Si el Board aprueba la solicitud, los cambios se integran al nivel estático.

Cómo realizar un cambio:

  • Obtener la última versión en el nivel controlado del elemento de configuración a cambiar.
  • Realizar los cambios en un lugar separado. Si se está usando git, crear una rama nueva para los cambios.

Cómo pasar de nivel dinámico a nivel controlado:

  • Hacer una solicitud de cambio para el elemento de configuración.
  • Esta solicitud debe incluir una descripción de los cambios que se realizaron y debe asignar revisores. Si estás usando git, hacer un pull request de la nueva rama hacia develop.
  • Los cambios deben pasar por una revisión y ser aprobados antes de integrarse con el nivel controlado

Cómo pasar de nivel controlado a nivel estático:

  • Los cambios a nivel controlado deben probarse
  • Al final de la iteración, hacer una solicitud de cambio para pasar lo que está en nivel controlado a nivel estático
  • El Configuration Control Board se reúne para revisar los cambios de manera exhaustiva
  • Si los cambios se aprueban, se integran al nivel estático

Cómo ver qué versión de un archivo está en cada nivel:

  • Si se está usando git, acceder al archivo en el repositorio
  • Dar click al dropdown "Branch: <nombre_branch>" y elegir una rama:
  • Si se desea ver la versión más reciente del archivo en el nivel estático, elegir la rama "master"
  • Si se desea la ver la versión más reciente del archivo en el nivel controlado, elegir la rama "develop"

El Configuration Control Board

El comité se encarga de revisar exhaustivamente los cambios propuestos al nivel estático. Este comité se compone de 5 miembros del departamento que tengan conocimiento del área de gestión de la configuración y ejerzan roles relacionados con la gestión del proyecto.
