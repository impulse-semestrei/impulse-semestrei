Guía de Preparación del Ambiente de Producción

Objetivo

Acceder al ambiente de producción de manera sencilla y dejarlo corriendo con la última versión desarrollada.

Pasos

  1. Acceder al cmd

  1. Insertar comando “ssh usuario@ipservidor”
  2. Insertar contraseña
  3. Viajar hasta la carpeta del proyecto
  4. Obtener última versión de la versión de test de la aplicación. Esto se puede hacer con el siguiente comando:

$ git pull origin test

  1. Realizar migraciones definidas. En caso de que haya conflictos con las migraciones, seguir las convenciones para conflictos con migraciones
  2. Ejecutar los tests del sistema con el siguiente comando:

$ py manage.py test

  1. Reiniciar nginx

sudo systemctl daemon-reload

sudo systemctl restart gunicorn

sudo systemctl restart nginx

  1. Hacer commit de cambios realizados desde que se obtuvo la última versión de test
